모달(Modal) 팝업과 모달리스(Modeless) 팝업 비교하기

1개요

웹스퀘어의 모달(Modal) 팝업과 모달리스(Modeless) 팝업을 비교할 수 있는 예제입니다.
$p.openPopup API의 옵션 중 "modal" 속성을 지정하여 모달 기능의 사용 여부를 결정합니다.
모달 기능은 팝업을 호출한 페이지(창)에 레이어가 생성되며 제어할 수 없습니다.

P00057

2구현된 기능

  • 모달 browserPopup 호출하기

  • 모달리스 browserPopup 호출하기

  • 모달 wframePopup 호출하기

  • 모달리스 wframePopup 호출하기

  • 모달 iframePopup 호출하기

  • 모달리스 iframePopup 호출하기

3예제 테스트 방법

화면에 버튼을 클릭하여 팝업의 형태를 비교합니다.

  • 버튼 browserPopup - modal : 모달 형식의 browser 팝업

  • 버튼 browserPopup - modeless : 모달리스 형식의 browser 팝업

  • 버튼 wframePopup - modal : 모달 형식의 wframe 팝업

  • 버튼 wframePopup - modeless : 모달리스 형식의 wframe 팝업

  • 버튼 iframePopup - modal : 모달 형식의 iframe 팝업 (wframe 팝업과 출력된 모양은 동일하지만 컨텐츠가 iframe에서 실행됩니다.)

  • 버튼 iframePopup - modeless 모달리스 형식의 iframe 팝업 (wframe 팝업과 출력된 모양은 동일하지만 컨텐츠가 iframe에서 실행됩니다.)

4구현 예시

아래의 두가지 속성값이 따라 팝업의 형태가 변경됩니다.

type : [default : "browserPopup", "iframePopup", "wframePopup"] 팝업의 종류.
modal : [default : "false", "true"] 모달 기능 사용 여부.
아래는 type 속성의 설정 값에 따른 설명입니다.

browserPopup : 독립된 별도의 창으로 생성되는 팝업입니다. 내부적으로 window.open을 통해 생성되는 팝업입니다. 프로세스가 분리되어 완전히 독립적으로 실행됩니다. 부모에 영향 받지 않고 팝업 위치를 자유롭게 움직일 수 있습니다.

iframePopup : iframe으로 생성되는 팝업입니다. 부모 화면에 종속적으로 생성됩니다. wframe 팝업과 출력된 모양은 동일하지만 컨텐츠가 iframe에서 실행됩니다.

wframePopup : wframe으로 생성되는 팝업입니다. 부모 화면에 종속적으로 생성됩니다.

4.1Modal Browser 팝업 생성하기

var jsnOptions = { 
    id : "popup_exam1_1", 
    type : "browserPopup", //필수 고정
    width: "300px", 
    height: "180px", 
    popupName : "Modal browserPopup", 
    modal : "true" //필수 고정
};

$p.openPopup("/page/P00057P01.xml",  jsnOptions );
